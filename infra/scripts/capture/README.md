# Screenshot capture pipeline — Phase 4L

Deterministic doc screenshots. Every image under
`assets/screenshots/` is *generated*, not hand-captured — so it can
be regenerated by anyone, on CI, after any visual change, with no
desktop and no manual cropping.

## How it works

`_render.py` runs Qt with the `offscreen` platform plugin: real
widget trees, no display. It loads a host sans-serif into the
(otherwise empty) offscreen font database so text renders as glyphs
rather than tofu, then `grab()`s a widget to a 2× PNG.

Same widgets in → same pixels out. The capture is as deterministic
as the engine it documents.

## Scripts

| Script | Renders | Output |
|---|---|---|
| `capture_launcher.py` | the Phase 4K digest — recovery, investigations, resurfacing, trust — plus loading / empty / offline / first-week states | `launcher-*.png` |
| `capture_recovery.py` | the recovery card alone — resting and keyboard-focused | `recovery-card*.png` |

Run all of them:

```bash
python infra/scripts/capture/capture_launcher.py
python infra/scripts/capture/capture_recovery.py
```

## Pending captures

Two surfaces are not yet automated, because each needs its own host
rather than the offscreen-Qt harness:

- **`capture_settings.py`** — the launcher Settings dialog. It is a
  real Qt dialog, so it can use `_render.py` directly; it just needs
  a `Config` fixture wired up.
- **`capture_extension.py`** — the extension popup is a built web
  page (`apps/extension/popup/`), so it needs a headless browser
  (Playwright) rather than Qt. The popup already supports
  storybook-style states via `?state=`, which is the capture input.

Both are tracked in [`PUBLIC_ALPHA.md`](../../../docs/founder/PUBLIC_ALPHA.md)
under known limitations.

## Conventions

- Output is always 2× for crisp doc embedding.
- Names are stable (`launcher-digest`, `recovery-card`, …) so doc
  references never break across regenerations.
- Demo content is fixed (the WebSocket-debugging fixture from
  `demo_seed.py`) so captures stay byte-stable.
