# Changelog

Recall's full, phase-by-phase changelog lives at
[`docs/release/CHANGELOG.md`](docs/release/CHANGELOG.md). The
single root file is kept here as a convention for tools that
look for a top-level `CHANGELOG.md`; the source of truth is the
release-folder version, alongside the
[`VERSIONING.md`](docs/release/VERSIONING.md) and
[`RELEASE.md`](docs/release/RELEASE.md) it pairs with.

See [`docs/release/CHANGELOG.md`](docs/release/CHANGELOG.md) for
the chronological record.
