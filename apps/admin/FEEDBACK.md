# Feedback inbox

Every piece of feedback, logged by hand. There is no in-app feedback
widget that phones home — feedback reaches here because a person
*sent* it (cohort thread, GitHub issue, email) and the founder
typed it in.

Newest first. One entry per item. Keep the user's words; resist
paraphrasing into something more flattering.

## Categories

| Tag | Means |
|---|---|
| `pain` | something was harder than it should have been |
| `bug` | something is broken |
| `confusing` | a moment the user did not understand |
| `feature` | a request for something new |
| `trust` | a doubt about privacy, data, or honesty |

## Entry format

```
### YYYY-MM-DD · [tag] · cohort:<id or "public">
What they said (their words).
→ Founder note: what it means / what it changes / linked issue.
```
