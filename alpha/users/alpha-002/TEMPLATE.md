# alpha-002 — second cohort (queued)

Mirrors [`alpha-001/TEMPLATE.md`](../alpha-001/TEMPLATE.md). Same
schema, same per-tester folder layout, same boundary.

The cohort opens **only when alpha-001 has at least three closed
week-one returns** (per the founder rule documented in
[`alpha/alpha_report.md`](../../alpha_report.md) — *"a cohort
closes when this file has at least three filled rows in each
section. Three is the floor below which the answers are anecdote,
not evidence."*).

Until then this folder stays empty save for the `TEMPLATE.md`.

The runbook is the same as alpha-001 — see
[`ALPHA_001_RUNBOOK.md`](../../ALPHA_001_RUNBOOK.md). What
differs between alpha-001 and alpha-002 is the *expected
distribution* of personas, which the founder rebalances based on
what alpha-001 surfaced (e.g. if the *student* persona produced
the strongest signal, alpha-002 may over-index there).
