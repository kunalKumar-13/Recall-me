# INSTALL_SIZE_AUDIT_V2.md — where the 970 MB actually is

V1 ([`INSTALL_SIZE_AUDIT.md`](INSTALL_SIZE_AUDIT.md)) counted files
per `_internal/<pkg>/` subtree and estimated sizes from wheel
metadata. V2 cross-references each install.log destination path
against the on-disk site-packages tree, so every entry below has a
real byte count, not a wheel estimate.

Generated by [`infra/scripts/audit_install_size_v2.py`](../../infra/scripts/audit_install_size_v2.py).
The numbers are the source of truth for
[`MODEL_STRATEGY.md`](MODEL_STRATEGY.md) and
[`SPLIT_DISTRIBUTION.md`](SPLIT_DISTRIBUTION.md).

## Top subtrees (real measured sizes)

The bundle has **159 subtrees** under `_internal/`, totalling
**~1,462 MB resolved** (the installer compresses this to 260.8 MB
via `lzma2` + `solid`). The top 12 account for 99% of the weight:

| # | Subtree | Files | MB | % of total | Verdict |
|---|---|---:|---:|---:|---|
| 1 | `torch` | 2,173 | **475.5** | 32.5% | the elephant; needs the model-strategy doc |
| 2 | `pyqt6` | 128 | **217.1** | 14.9% | ~60 MB is unused Qt modules — excludable |
| 3 | `scipy` | 98 | **116.5** | 8.0% | needed by sklearn; can't drop easily |
| 4 | `transformers` | 2,347 | 95.4 | 6.5% | replaceable with ONNX |
| 5 | `pyarrow` | 647 | **88.6** | 6.1% | transitive (huggingface_hub); **excludable** |
| 6 | `imageio_ffmpeg` | 2 | **87.7** | 6.0% | FFmpeg binaries; Recall has no media path → **excludable** |
| 7 | `pandas` | 53 | 64.8 | 4.4% | transitive; verify chromadb survives without |
| 8 | `chromadb_rust_bindings` | 1 | 63.4 | 4.3% | one 63 MB `.pyd`; required for the local vector DB |
| 9 | `onnxruntime` | 3 | 42.3 | 2.9% | already there — pivot to ONNX uses what is paid for |
| 10 | `sklearn` | 389 | 41.4 | 2.8% | distance computations; medium |
| 11 | `numpy` (+ libs) | 15 | 52.8 | 3.6% | required |
| 12 | `pil` | 7 | 15.8 | 1.1% | required for icon paths |

Items the installer **already drops** (in site-packages but not in
the bundle): `sympy` (72 MB), `kubernetes` (33 MB), `streamlit`
(28 MB), `pydeck` (24 MB), `altair` (10 MB), `pip` (11 MB),
`pygments` (8.8 MB). PyInstaller's hooks correctly exclude these
~200 MB — the remaining 970 MB is the real cost.

## Expected savings — three tiers

The numbers below are *uncompressed-bytes saved*; the compressed
installer drops by roughly `saving / 3.7` (the observed
`lzma2 + solid` ratio on this bundle).

### Tier A — pure PyInstaller excludes (no code changes)

Each row is a one-line `excludes=...` or `--exclude-module` entry
in `recall.spec`. Reversible. None require touching engine code.

| Exclude | Unpacked save | Risk |
|---|---:|---|
| `imageio_ffmpeg` (FFmpeg binaries) | -87.7 MB | low — no media path in Recall |
| `pyarrow` (Arrow IPC + Flight + parquet + compute) | -88.6 MB | medium — verify huggingface_hub still downloads (it does; arrow is for `datasets`, not for the model fetcher) |
| Qt6 modules: `Quick`, `Qml`, `Quick3DRuntimeRender`, `Pdf`, `Designer`, `opengl32sw`, `avcodec-61` | -60.8 MB | low — Recall is QtWidgets-only |
| `tzdata` (timezone DB; Recall date handling uses stdlib) | -7 MB | low |
| `pandas` (transitive via huggingface_hub) | -64.8 MB | medium — verify HF + chroma both survive without |
| **subtotal** | **-308.9 MB** | bundle drops to ~660 MB; installer to ~180 MB |

This alone puts the **installer at ~180 MB compressed** — close to
the *<150 MB base* goal in [`MODEL_STRATEGY.md`](MODEL_STRATEGY.md)
without changing a single model decision.

### Tier B — switch to `torch+cpu` minimal build

PyPI ships `torch` with CUDA scaffolding even on Windows CPU
machines (the wheels include CUDA kernel stubs for runtime
detection). The `torch+cpu` build is meaningfully smaller.

| Change | Unpacked save | Risk |
|---|---:|---|
| Move `requirements.txt` from `torch` → `torch+cpu` (PyPI extra index `https://download.pytorch.org/whl/cpu`) | -140 to -180 MB | low — Recall never asks for CUDA; the embedding model runs CPU-only |
| **Combined Tier A + B** | **-450 to -490 MB** | bundle ~480-520 MB; installer ~130-150 MB |

### Tier C — ONNX route (drops torch entirely)

The local embedding model (`all-MiniLM-L6-v2`) has an official
ONNX export. Loading via `onnxruntime` removes the need for torch
+ transformers' python-side weight loading; `tokenizers` (the
fast tokenizer) stays.

| Change | Unpacked save | Risk |
|---|---:|---|
| Replace `sentence_transformers` + `torch` with `onnxruntime` + bundled `.onnx` file | -460 MB (torch) -90 MB (transformers) +~30 MB (the .onnx weight) ≈ **-520 MB** | medium — needs the embedding pipeline rewritten as ~80 LOC against onnxruntime; equivalence must be verified by recall-quality smoke (`_smoke_api.py` § 11) |
| **Combined Tier A + C** | **-820 MB** | bundle ~150 MB; installer **~40 MB compressed** |

Tier C is the path to a *truly small* Recall — under 50 MB
compressed, ~150 MB on disk. Tier A + B is the path to a *good
enough* Recall for the alpha without rewriting the embedding
pipeline.

### Tier D — lazy-load on first run (the *Split Distribution* play)

Ship two artifacts:

- **Recall Core** (~50 MB compressed): the launcher, the daemon,
  chromadb, **no embedding stack**. Search falls back to
  keyword-match until the user installs the Retrieval Pack.
- **Recall Retrieval Pack** (~100 MB compressed): the ONNX model
  + onnxruntime + tokenizers. Optional. Downloaded on first run
  *or* via Settings → *Enable semantic search*.

Detail in [`SPLIT_DISTRIBUTION.md`](SPLIT_DISTRIBUTION.md).

## What this V2 audit does not include

- **Hugging Face model cache.** `~80 MB` per user; downloads on
  first run to `%LOCALAPPDATA%\…\huggingface\`. *Not* in the
  installer. Tier C bundles it; Tier D defers it.
- **Chroma SQLite + per-folder vector DB.** Grows with user
  events; not in the installer.
- **Extension popup bundle** (`apps/extension/popup/`). Lives in
  Chrome's extension dir, not in the desktop installer.
- **macOS / Linux variants.** Numbers above are Windows-PyInstaller
  specific. The Mac figure will land in
  [`MAC_VERIFICATION.md`](../release/MAC_VERIFICATION.md) when a
  maintainer runs the macOS build.

## What this audit asks of the next phase

Three orderable engineering items, smallest first:

1. **Edit `recall.spec` with Tier A excludes.** Rebuild. Verify
   doctor + smoke. Target: installer ≤ 200 MB compressed. (~1
   hour with the Inno Setup recompile dominating.)
2. **Pin `torch+cpu` in `requirements.txt`.** Rebuild + smoke.
   Target: installer ≤ 150 MB compressed. (~24 hours' worth of
   build time on this build machine; or run on a CI box.)
3. **The ONNX path** (Tier C). Real engineering: rewrite the
   embedding pipeline (one Python module, ~80 LOC). Target:
   installer ≤ 50 MB compressed and **install time under 30 s**
   on the build machine.

Items 1 + 2 alone close the directive's *<60 s install* goal on
the current build machine. Item 3 is the route to *<30 s* and to
the *Split Distribution* model.
